# Atlas — Bot (Athena AI First Responder)

## What it does

Athena intercepts every new support ticket and attempts to answer from the knowledge base before a human agent is involved. If it can't answer confidently, it escalates to the on-shift primary agent.

Goals:
- Median first-response time drops from minutes/hours to ~5s for answerable tickets
- Human agents focus on novel problems rather than repeating documentation
- Every escalation is a knowledge-gap signal

---

## Tech stack

| Layer | Technology |
|---|---|
| Queue | pg-boss (`bot:respond-to-ticket`) |
| Vector store | pgvector (HNSW index) in the existing Postgres |
| Sparse search | pg_trgm GIN index |
| LLM | gemini-2.5-flash-lite (chat) + text-embedding-004 (embedding) |
| Config | `AppConfig.bot*` + `AppConfig.kb*` fields |
| Observability | `BotInteraction` table (full audit trail per ticket) |

---

## Data model

| Model | Purpose |
|---|---|
| `KnowledgeSource` | One row per crawled/manually-added URL. Tracks status, contentHash (dedup), chunk count. |
| `KnowledgeChunk` | Split + embedded sub-section. Stores `headingPath`, `anchor`, `deepUrl`, 768-d `embedding` (pgvector). |
| `BotInteraction` | Audit trail: which chunks were retrieved, retrieval score, LLM confidence, decision, cost, latency, citations. |
| `Shift` | Weekly on-call windows. Used by `ShiftResolverService` to pick the escalation target. |

New `AppConfig` fields: `botProvider`, `botApiKeyEnc`, `botModelChat`, `botFallbackAgentId`, `kbRootUrl`, `kbCrawlStatus`, `kbCrawlPagesSeen`, `kbCrawlPagesIndexed`, `kbCrawlError`, `kbLastRecrawledAt`, `timezone`.

Removed fields (hardcoded in code): `botEnabled` (always on), `botModelEmbedding` (always `gemini-embedding-001`), `botRetrievalThreshold` (0.01 — RRF scale), `botConfidenceThreshold` (0.7), `botName` ("Athena"), `botAvatarUrl`.

New `Message` field: `authorBotName` — when non-null, this message was generated by Athena.

New `AiUsage` field: `userId` — enables per-user cost roll-ups.

New `AiOperation` enum values: `ATHENA_EMBED`, `ATHENA_GENERATE`, `KB_CONTEXTUAL_SUMMARY`.

New `AgentRole` values: `PRIMARY_AGENT`, `SECONDARY_AGENT` (replaces old `AGENT`).

## RAG pipeline

```
New ticket created
        │
        ▼
enqueueBotRespond (pg-boss)
        │
        ▼
RespondToNewTicketWorker
        │
        ├─ BotService.respondTo(ticketId)
        │       │
        │       ├─ 1. Check idempotency (BotInteraction already exists?)
        │       │
        │       ├─ 2. GeneratorService.embed(question) → 768-d vector
        │       │
        │       ├─ 3. RetrievalService.retrieve()
        │       │       ├─ Dense: pgvector HNSW (top-50)
        │       │       ├─ Sparse: pg_trgm similarity (top-50)
        │       │       └─ Reciprocal Rank Fusion (k=60) → top-20 → top-5
        │       │
        │       ├─ Gate 1: maxScore < 0.55 (DENSE_THRESHOLD) → ESCALATE
        │       │
        │       ├─ 4. GeneratorService.generateAnswer(question, top5)
        │       │       └─ Gemini JSON mode → {answer, citations[], confidence, can_answer, reasoning}
        │       │
        │       ├─ Gate 2: !can_answer → ESCALATE
        │       ├─ Gate 3: confidence < 0.7 (CONFIDENCE_THRESHOLD) → ESCALATE
        │       ├─ Gate 4: citations.length === 0 → ESCALATE (hallucination guard)
        │       ├─ Gate 5: any citation outside kbRootUrl origin → ESCALATE (anti-fabrication)
        │       │
        │       ├─ ANSWER: appendSource() adds "Learn more:" link from citations[0]
        │       │          → create Message(authorBotName), status → WAITING, send email
        │       └─ ESCALATE: ShiftResolverService → assign ticket, create SYSTEM_EVENT
        │
        └─ Write BotInteraction audit row (always)
```

## Knowledge base pipeline

```
POST /kb/crawl/start
        │
        ▼
enqueueKbCrawl (pg-boss)
        │
        ▼
CrawlAndIndexWorker
        │
        ├─ CrawlerService.crawl(rootUrl)
        │       ├─ Tier 1: sitemap.xml / sitemap_index.xml
        │       ├─ Tier 2: BFS link discovery (max depth 5, max 500 pages)
        │       └─ Tier 3: Playwright render (JS SPAs, if body < 200 chars)
        │
        └─ For each page → IndexerService.indexPage(url)
                │
                ├─ fetch HTML (or skip if contentHash unchanged)
                ├─ ChunkerService.chunk(html, url)
                │       ├─ Turndown HTML → Markdown
                │       ├─ Split on H2/H3 boundaries
                │       ├─ Merge undersized (<200 tokens) with sibling
                │       ├─ Split oversized (>800 tokens) at paragraph boundary
                │       └─ Never split mid-code-block
                │
                ├─ ContextBuilderService.buildContextHeader(title, body)
                │       └─ 1 Gemini Flash call → 50-token doc summary
                │
                ├─ EmbeddingService.embedChunks(texts)
                │       └─ text-embedding-004, batched 100/call → 768-d vectors
                │
                └─ Prisma transaction: delete old chunks → insert new chunks with embedding
```

## Shift-based routing

`ShiftResolverService.currentPrimaryAgent(now: Date)` — deterministic, pure logic.

Resolution chain:
1. Convert `now` to `AppConfig.timezone`
2. Find active `Shift` rows matching `(dayOfWeek, minuteOfDay)` — handles overnight wrap
3. Multiple matches → round-robin (smallest `lastAssignedAt` wins)
4. No match → `AppConfig.botFallbackAgentId`
5. Still null → first active `PRIMARY_AGENT`
6. Still null → first active `ADMIN`
7. Still null → log error, return null (ticket unassigned)

## Settings UI

| Page | Path |
|---|---|
| AI Assistant | `/settings/ai-assistant` |
| Shifts | `/settings/agents` (Shifts section at the bottom of the Agents page; `/settings/shifts` now redirects here) |

**AI Assistant page** — two cards:
1. **Provider** — Gemini API key (set/update), provider selector, chat model override
2. **Knowledge Base** — root URL, crawl status/progress, pages table with per-row Re-index/Delete, "Add page manually" input, "Clear index" confirm

Removed from UI (hardcoded): embedding model, test-connection button, enable/disable toggle, retrieval/confidence threshold sliders, bot name. **Fallback agent** moved to Settings → Agents page.

**Shifts** — now a section at the bottom of the **Agents** settings page (`ShiftsSection` component), keeping on-call routing config next to the agents it routes to. Timezone selector + list of shifts with add/delete/toggle-active controls. The standalone `/settings/shifts` route is a redirect to `/settings/agents` for back-compat, and the separate Calendar rail icon was removed. The agent picker filters for `PRIMARY_AGENT`/`ADMIN`; the agents invite/role-change UI was fixed to use the real `AgentRole` values (was sending the non-existent `AGENT`, which Prisma rejected — silently blocking new primary agents from being schedulable).

## Cost model (per-ticket ballpark)

| Step | Cost |
|---|---|
| Embed query (~50 tokens) | ~$0.0000013 |
| Dense + sparse retrieval | $0 (SQL) |
| Generation (~2000 in + 200 out tokens) | ~$0.0002 |
| **Total per answered ticket** | **~$0.0002** |
| Full corpus index (5k chunks × 500 tokens) | ~$0.06 one-time |
| Weekly re-crawl with content-hash dedup | ~$0.05/month |

## Decisions

| Decision | Why |
|---|---|
| pgvector in existing Postgres | Zero new infra; corpus is small (hundreds–thousands of pages) |
| Async via pg-boss | Ticket creation stays fast; bot runs in background |
| Hybrid retrieval (dense + sparse + RRF) | Pure vector search misses exact product names / error codes |
| Contextual retrieval (Anthropic 2024 pattern) | Doc-level summary prepended to every chunk lifts recall 35–50% at trivial cost |
| AgentRole rename AGENT → PRIMARY_AGENT | Semantically correct; every previously-assignable agent stays assignable as first responder |
| Phase 1 defers reranker | RRF over top-50 is sufficient for help-center size; add if quality demands it |
| Phase 1 defers Playwright for JS SPAs | Fallback already stubbed; activate when a JS-rendered docs platform is onboarded |
