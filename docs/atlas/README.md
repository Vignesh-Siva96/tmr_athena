## System

```mermaid
flowchart LR
  Customer((Customer))
  Agent((Agent))

  Customer -->|portal form| Portal[Portal · Next.js]
  Customer -.->|sends email| Gmail[(Support inbox<br/>e.g. Gmail)]
  Agent --> Bridge[Bridge · Next.js]

  Portal --> API[API · NestJS]
  Bridge --> API

  Gmail -.->|IMAP IDLE| API
  API -.->|SMTP| Gmail

  API --> Postgres[(Postgres)]
  API --> MinIO[(MinIO<br/>attachments)]
  API --> Queue[(pg-boss<br/>jobs in Postgres)]
  API --> GitHub[(GitHub<br/>OAuth + Issues + Webhook)]

  Gmail -.->|delivers reply| Customer
```

## How to use this atlas

- **"Where does feature X live?"** → its `<feature>.md`, "Key files" section.
- **"What endpoints exist?"** → [_generated/api-routes.md](_generated/api-routes.md).
- **"What's the schema?"** → [_generated/erd.md](_generated/erd.md).
- **"How does inbound email actually work?"** → [email.md](email.md), "Inbound flow" section.
- **"When was X changed?"** → still [STATE.md](../../STATE.md). This atlas describes *now*, not *history*.
